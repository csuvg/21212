Explicación de los componentes

Componentes principales en React

Para esta ocación, se utilizarán los siguientes componentes principales en React:

  • Pokemon: componente para la carga de pokemones desde el API, como también de su despliegue en la página
  • PokeDetails: detalles de los Pokemon como su HP, fuerza, habilidad, etc.
  • Searcher: barra de búsqueda

Estructura del código y organización de carpetas

La estructura del código se organizará de la siguiente manera:

  • src
    • components
        • Pokemon.jsx
        • Pokemon.css
        • PokeDetails.jsx
        • PokeDetails.css
        • Searcher.jsx
        • Searcher.css
    • hooks
      • pokeHook.js/li>
    • App.jsx

Explicación del consumo de datos mediante la API

¿Cómo se harán las llamadas al PokeAPI?

Las llamadas se hará mediante el uso de fetch y funciones asíncronas, las cuales obtendrán por medio de GET la respuesta del API en formato JSON.

¿Cómo se manejarán los datos, una vez recibidos?

Los datos se manejarán por medio de estados y hooks. Los primeros se actualizarán cada vez que se haga una llamada al API.

Análisis del problema y solución propuesta

Desafíos a enfrentar

Los principales desafíos serían:

  • Manejo de estados
  • Carga desde el API
  • Creación de tarjetas
  • Manejo de detalles para cada Pokemon

Estructura de la interfaz

La solución final tendrá de inicio una barra de búsqueda con un botón donde se puedan buscar los pokemones por su nombre o ID. Abajo estarán las tarjetas y al hacer click en alguna, se desplegará el detalle del Pokemon dentro de un modal.

Navegación e interacción de los datos

La navegación se hará por medio de la barra de búsqueda y la interacción será por medio de las tarjetas y el modal con los detalles desplegados según cada Pokemon.
